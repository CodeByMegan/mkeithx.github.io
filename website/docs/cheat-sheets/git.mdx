## Using `git switch`

<details>
<summary>Show</summary>

<div>
<div>`git switch` enhances branch management in Git, making it easier to switch between, create, and merge branches.</div>

1. **Switch to an Existing Branch**

```bash
git switch branch-name
```

2. **Create and Switch to a New Branch**

```bash
git switch -c new-branch-name
```

3. **Work on Your Branch**

```bash
git add .
```

```bash
git commit -m "Describe your changes"
```

4. **Push Your Branch to Remote**

```bash
git push -u origin new-branch-name
```

5. **Merge Changes from Another Branch**

```bash
git switch your-branch
```

```bash
git merge main
```

6. **Resolve Conflicts (if any)**

```bash
git add .
```

```bash
git commit -m "Resolved merge conflicts"
```

7. **Switch Back to Another Branch**

```bash
git switch main
```

8. **Clean Up**

```bash
git branch -d new-feature
```

```bash
git push origin --delete new-feature
```

</div>

</details>

### Extended

<details>
<summary>Show</summary>

```js title=".gitconfig"

# First things first, always set up your user info.
// highlight-start

[user]
name = my_username
email = my_email@example.com

// highlight-end

# Display colours in the console
[color]
ui = auto
branch = auto
status = auto

# Always rebase when using `git pull`, except for `main, master, develop` branches.
[branch]
autosetuprebase = always

[branch "main"]
rebase = false

[branch "master"]
rebase = false

[branch "develop"]
rebase = false



# Automatically set up remote tracking branches for all new local branches.



[push]
autoSetupRemote = true

# Important setting in windows that allows working on both Windows and cross-platform projects
[core]
eol = native
autocrlf = input

# Common typos and aliases for long commands
[alias]
chekcout = checkout
ocmmit = commit
statsu = status
sttaus = status
fa = fetch --all
pushf = push --force-with-lease
stsah = stash

# Common LFS Settings
[filter "lfs"]
smudge = git-lfs smudge -- %f
process = git-lfs filter-process
required = true
clean = git-lfs clean -- %f

# How private registries are handled
[credential "https://my_company_vcs.example.com"]
provider = generic
```

</details>
