# Get started

Before diving into tutorials, consider a few best practices. Creating a virtual environment like .venv keeps your projects organized. Follow along with these tips to ensure a smooth learning experience and effective implementation in your work.




## Creating a Virtual Environment in Python

When working on Python projects, it's a good practice to use a virtual environment to isolate dependencies and keep your project environment clean. Follow these steps to create a virtual environment:

1. Open your terminal or command prompt.

2. Navigate to your project's root directory using the `cd` command:


<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">
   
    ```bash
   cd path/to/your/project
```
  
  </TabItem>
  <TabItem value="unix" label="Unix">
  
  ```bash
   cd path/to/your/project
```
  
  </TabItem>
</Tabs>


3. Run the following command to create a virtual environment named `.venv`:


<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">

  ```bash
  py -m venv .venv
```
  
  </TabItem>
  <TabItem value="unix" label="Unix">
  
```bash
python3 -m venv .venv
```


  </TabItem>
</Tabs>

4. Activate the virtual environment.

<Tabs groupId="operating-systems">
  <TabItem value="win" label="Windows">

  ```sh
  .venv\Scripts\activate
```
  
  </TabItem>
  <TabItem value="unix" label="Unix">

  ```bash
  source .venv/bin/activate
  ```
  </TabItem>
</Tabs>
