## GitHub Actions Workflow

To further enhance my development process, I’ve set up a GitHub Actions workflow to automate testing and deployment. Here’s a quick overview of my workflow configuration:

```yaml
name: CI/CD Workflow

on:
  push:
    branches:
      - main
  pull_request:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout code
      uses: actions/checkout@v2

    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: '3.9'

    - name: Install dependencies
      run: |
        pip install -r requirements.txt

    - name: Run tests
      run: |
        pytest

    - name: Deploy
      if: github.ref == 'refs/heads/main'
      run: |
        # Your deployment script here
```

This workflow ensures that every push and pull request triggers tests, helping me maintain code quality and reliability.
